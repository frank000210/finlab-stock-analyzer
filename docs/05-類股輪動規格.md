# 類股輪動視覺化功能規格（Sector Rotation）

## 1. 目標

新增獨立頁面 `/rotation`，用歷史資料觀察「類股漲跌輪動順序」，且**不修改現有 `/graph` 視覺**。

主視覺為 RRG（Relative Rotation Graph）輪動時鐘；輔視覺為類股領先落後有向圖（Lead-Lag Graph）。

---

## 2. 範圍

### 2.1 In Scope
- 雙資料來源 universe：
  - `twse`：TWSE 官方類股指數（主）
  - `watchlist`：觀察池聚合等權類股（輔）
- 雙頻率：
  - `daily` 日頻
  - `weekly` 週頻
- API：`build / snapshot / timeline / ranking`
- 前端頁面：`RotationView.vue`
  - RRG 四象限散點 + 軌跡尾巴 + 日期播放
  - 領先落後有向圖
  - 輪動棒次條
  - 點選節點/邊右側明細

### 2.2 Out of Scope
- 不取代原本 `/graph`
- 不新增交易下單策略

---

## 3. 資料模型與來源

## 3.1 官方類股（TWSE）
- 來源：`MI_INDEX`（`type=IND`）
- 儲存集合：
  - `raw_sector_index`：`{sector_id, name, date, close, is_benchmark, updated_at}`
  - `raw_sector_index_days`：`{date, row_count, updated_at}`（含休市空白快取）
- Benchmark：`TAIEX`（發行量加權股價指數）

## 3.2 觀察池聚合
- 來源：
  - `raw_prices`（個股收盤）
  - `raw_industry`（個股產業）
- 方法：每檔先 normalize 到 base=100，再按產業做等權平均形成產業曲線
- Benchmark：觀察池全體等權指數

---

## 4. 計算規則

## 4.1 RRG 指標
- `RS = 100 * sector_close / benchmark_close`
- `RS-Ratio = 100 + zscore(RS, trailing_window)`
- `RS-Momentum = 100 + zscore(ROC(RS-Ratio, momentum_period), trailing_window)`

預設參數：
- `daily`：`window=50`, `momentum=10`, `max_lag=5`
- `weekly`：`window=10`, `momentum=4`, `max_lag=3`

## 4.2 象限定義
- `leading`：`RS-Ratio >= 100` 且 `RS-Momentum >= 100`
- `weakening`：`RS-Ratio >= 100` 且 `RS-Momentum < 100`
- `lagging`：`RS-Ratio < 100` 且 `RS-Momentum < 100`
- `improving`：`RS-Ratio < 100` 且 `RS-Momentum >= 100`

## 4.3 接棒順序（Lead-Lag）
- 使用 RS-Ratio 差分序列做 lag correlation
- 方向：`src -> dst` 表示 src 領先 dst
- 邊屬性：
  - `lag`
  - `weight`
  - `abs_weight`
  - `directed=true`
- 邊門檻：`edge_threshold`（預設 `0.25`）

## 6. API 規格

## 6.1 POST `/api/v1/rotation/build`
- 用途：更新原始資料（官方類股或觀察池聚合）
- body：
```json
{
  "universe": "twse|watchlist",
  "symbols": ["2330", "2317"],
  "end": "2026-07-04",
  "lookback_days": 400
}
```

## 6.2 GET `/api/v1/rotation/snapshot`
- 查單日輪動快照
- query：`universe,freq,date,symbols,lookback_days,trail_length,edge_threshold`
- 回傳重點：
  - `points[]`（類股當前座標與象限）
  - `trails{}`（每個類股軌跡）
  - `lead_edges[]`（有向接棒邊）
  - `ranking[]`（輪動棒次）

## 6.3 GET `/api/v1/rotation/timeline`
- 查日期序列（播放用）
- query：`universe,freq,start,end,symbols,lookback_days`

## 6.4 GET `/api/v1/rotation/ranking`
- 查指定日期輪動排行
- query：`universe,freq,date,symbols,lookback_days`

## 7. 前端規格（`/rotation`）

## 7.1 控制項
- universe 切換：官方類股 / 觀察池聚合
- 頻率切換：日頻 / 週頻
- 日期區間
- lookback、尾巴長度、邊門檻
- Build（更新原始資料）/ Reload（重算）
- 日期滑桿 + Play/Pause

## 7.2 視覺
- RRG 輪動時鐘：
  - 四象限底色
  - 點位 + 軌跡尾巴
  - 點選高亮
- 有向圖：
  - 類股節點（顏色 = 象限）
  - 箭頭邊（粗細 = `abs_weight`）
- 棒次條：
  - 類股排序 chip，點擊可回選節點

## 7.3 互動
- 點節點：右側顯示類股指標（RS-Ratio / RS-Momentum / angle / quadrant）
- 點邊：右側顯示接棒屬性（lag / weight / abs_weight）
- 日期切換自動重載 snapshot

---

## 8. 驗收條件

1. `/rotation` 頁面可正常載入與切換日/週頻
2. `twse` 與 `watchlist` 兩種 universe 都能產生資料
3. timeline 播放可驅動 RRG 與有向圖同步更新
4. 點節點/邊右欄資訊正確
5. API `build/snapshot/timeline/ranking` 回傳 `success=true`

---

## 9. 風險與對策

- TWSE 歷史回補量大：使用 `raw_sector_index_days` 快取，避免重抓休市日
- 觀察池樣本過少：前端保留空狀態與錯誤提示
- 參數敏感：保留 `lookback/trail/threshold` 可調，不硬編固定值
