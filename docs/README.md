# 開發文件總覽

本資料夾是一套**可複製的網站開發流程**。開發任何新網站都能照這套文件跑：
先寫規格 → 用提示詞驅動 AI → 開發 → check-in → 部署 → 線上驗證 → 迭代。

| 檔案 | 用途 | 何時看 |
|---|---|---|
| `00-開發流程.md` | 可複製 SOP：10 階段流程、MCP/工具、部署踩雷、檢查清單 | 開新專案先讀 |
| `01-規格書範本.md` | 規格書範本（由下而上，含 finlab 已填範例） | 規劃需求時複製填寫 |
| `02-AI提示詞範本.md` | 給 AI 的系統/任務/部署/迭代提示詞 | 驅動 AI 開發時 |
| `03-技術架構.md` | 本專案實際前後端技術架構與資料流 | 理解系統 / 接手時 |
| `04-開發指引.md` | 實務開發約定、部署、線上驗證、除錯、頁面觀測重點規則 | 動手寫程式時 |
| `05-類股輪動規格.md` | 類股輪動（Rotation）功能規格 | 開發/理解輪動相關功能時 |
| `06-頁面觀測重點與改動清單.md` | 每頁的觀測重點、現況落差、D3 gallery 圖表清單與完成狀態 | 新增/修改任一頁面前 |
| `07-UIUX交接摘要.md` | 給 Claude Design 用的設計 token、頁面清單、視覺待辦 | 交接 UI/UX 設計工作時 |

## 套用到新專案的最短路徑

1. 複製 `01-規格書範本.md`，回填需求澄清答案與各層模組。
2. 把 `02-AI提示詞範本.md` 的 A 段系統提示詞 + B 段任務提示詞貼給 AI。
3. 依 `00-開發流程.md` 跑「開發 → 驗證 → 部署 → 實測」迴圈。
4. 用 `04-開發指引.md` 的 Definition of Done 收尾每個增量。

> 核心心法：**每輪只做一個可驗證的小增量，做完一定 build → push → deploy → 線上驗證再進下一輪。**
