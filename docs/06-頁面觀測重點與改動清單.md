# 頁面觀測重點與改動清單

> 搭配 `04-開發指引.md` 使用。本文件定義「每個頁面該回答什麼問題」，以及既有頁面目前離這個目標還差什麼。

---

## 一、原則

1. **每個頁面只對應一個觀測重點**（一句話講清楚：這頁是要讓使用者看懂什麼）。首頁例外，不受此規範限制。
2. **先寫觀測重點，再設計版面**：新增頁面時，先寫好這句話，再決定要放哪些圖表/卡片；不要先堆圖表再回頭找理由。
3. **每個頁面頂部必須用 `PageFocusBanner` 元件顯示這句話**（`frontend/src/components/PageFocusBanner.vue`），放在頁面根 `<div>` 的第一個子元素：
   ```html
   <div class="xxx-page">
     <PageFocusBanner text="這頁的觀測重點一句話" />
     ...
   </div>
   ```
4. **加新圖表前先自問**：這個圖表是在回答本頁的觀測重點，還是另一個頁面的問題？如果是後者，應該放到對應頁面，而不是塞進目前這頁。
5. **改動時同步更新本文件**：頁面用途改變（觀測重點改了）或新增/移除頁面時，回來更新下面的總表，避免文件與程式碼脫節。

## 三、現況比對與待改清單

以下是重新檢視每個頁面「現有內容」對照「觀測重點」後，發現尚未完全對齊、需要補強或調整的項目。沒有列出的頁面代表現況內容已經符合觀測重點，暫不需要改動。

### OverviewView（觀測重點：綜合健康度總覽）
- 現況：雷達圖 + 各維度（季節性/領先落後/主力/社群/基本面）摘要卡片，符合「總覽」定位。
- **✅ 已完成**：每日漲跌熱力圖（Treemap，參考 D3 gallery - Treemap），放在雷達圖之前。後端 `GET /api/v1/rotation/heatmap`（`analysis/sector_rotation.py::get_daily_heatmap`），前端 `d3.hierarchy` + `d3.treemap`。方塊大小＝|漲跌幅|、顏色＝漲跌方向，資料為官方類股指數今天對昨天的收盤變化。

### GraphView（觀測重點：關聯強度與網絡結構）
- 現況：力導向網絡圖，看得出「跟誰連動」的結構，但看不出精確的相關係數數值。
- **✅ 已完成**：新增「相關矩陣熱力圖」檢視模式（參考 D3 gallery - Cluster dendrogram / 相關係數熱力圖），可用既有的 view-mode 切換器選擇，作為網絡圖的另一個檢視角度，沿用同一組 `snapshotNodes`/`activeEdges` 資料，無需新後端端點。

### AnalysisView（觀測重點：技術面是否支持進出場時機）
- 現況：K線+均線、RSI/MACD/KD、布林通道、ADX/ATR/OBV 已完整覆蓋動能與波動面向。
- **✅ 已完成**：新增成交量分佈圖（Volume Profile，參考 D3 gallery - Contour/Hexbin），疊在每日漲跌日曆熱力圖之下。因為目前只有日 OHLCV、沒有逐筆成交資料，採用「當日成交量平均分攤到當日高低價區間」的近似算法，畫面下方明確標註為近似估算、非真實逐筆分價量表。
- **✅ 已完成**：新增每日漲跌日曆熱力圖（Calendar，參考 D3 gallery - Calendar），GitHub 風格週曆排版，用個股每日漲跌幅上色，用來看長時間下哪些日子強、哪些弱。

### PublicDataView（觀測重點：判斷股價相對便宜或昂貴）
- **✅ 已完成**：本益比／股價淨值比河流圖（PE/PB Band，參考 D3 gallery 分類延伸），沿用既有 `GET /api/v1/analysis/{symbol}/fundamental?metrics=revenue,eps` 端點（後端本已提供 `eps_quarterly`，無需新增後端程式碼），前端計算 TTM EPS 滾動加總，再以歷史本益比的 20/40/60/80 百分位帶疊上實際股價。
- **✅ 已完成**：月營收年增率（YoY）柱狀圖疊營收走勢（參考 D3 gallery Bars 分類延伸），沿用同一 fundamental 端點的 `revenue_monthly`。
- 兩張圖已放在頁面最上方，既有的公告/配息/財務摘要維持在下半部作為輔助明細，符合「先回答觀測重點」的頁面設計原則。

### SeasonalView（觀測重點：季節性慣性）
- 現況：文字規律卡片 + 月報酬率熱力圖 + 各月統計卡片，已符合觀測重點。
- **✅ 已完成（加強版）**：新增各月報酬分布箱型圖（參考 D3 gallery - Box plot），沿用既有 `raw_monthly` 資料在前端算四分位數，不需要新的後端端點。熱力圖看「哪一年哪個月」，箱型圖補上「這個月歷年報酬的離散程度與離群值」，兩者互補。

### RotationView（觀測重點：類股輪動與領先/落後關係）
- 現況：RRG 象限圖 + 領先落後有向力導向圖 + 類股選擇器。

- **✅ 已完成（示意性質）**：AI 訊號 Sankey 分流圖（參考 D3 gallery - Sankey diagram）。本頁沒有真實資金流資料，因此改以 AI 訊號信心度為權重，示意訊號從「買入/賣出/觀望」分流至個股的相對比重；畫面下方明確標註這不是真實持倉或資金流向。因沙箱環境的 npm registry 無法安裝 `d3-sankey`，改用純 `d3.path` 手刻 ribbon 版面，未引入新依賴。
